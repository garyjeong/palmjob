# PalmJob - 기획서 요약 (PRD)

> 원본: `20251228 - 기획서.pdf`

---

## 1. 프로젝트 개요

**PalmJob**은 손바닥 사진을 분석하여 **이색 직업 1개를 추천**하고, **결과 카드 이미지**와 **해석 문장**을 제공하는 웹 서비스입니다.

| 항목 | 내용 |
|------|------|
| 서비스 성격 | 재미·공감·공유 (과학적 진단 아님) |
| 수익 모델 | 광고 기반 |
| 로그인 | 없음 (즉시 이용) |
| 결과 보관 | 30일 |

---

## 2. 핵심 원칙

- ✅ 연령 제한 없음
- ✅ 왼손/오른손 가이드 반드시 구분
- ⚠️ **손바닥 사진 절대 저장 안 함**
- ✅ 공유는 결과 랜딩 페이지 링크 중심
- ❌ 로그인/회원 기능 없음
- ⏰ 결과 랜딩 보관 기간: 30일

---

## 3. 서비스 목표

1. 업로드 → 결과 확인까지 **단순하고 빠른 완주**
2. 결과물이 **저장/공유할 만한 완성도** 보유
3. 공유 링크 유입 사용자의 **"나도 해보기" 전환**
4. **외부 생성 비용 최적화** 최우선

---

## 4. 타깃 사용자

- 운세/테스트/밈 콘텐츠 소비 및 공유 사용자
- 친구/연인과 비교하며 즐기는 사용자
- SNS/커뮤니티/메신저로 링크 공유를 자주 하는 사용자

---

## 5. 대표 시나리오

```
1. 사용자가 PalmJob에 접속
2. 왼손/오른손 선택 후 손바닥 사진 업로드
3. 결과 확인 (직업 + 결과 카드 + 해석 문장)
4. 공유하기 → 결과 랜딩 링크 전달
5. 링크 수신자가 결과 확인 후 "나도 해보기"로 참여
```

---

## 6. 화면 구성 (4개 화면)

### 6.1 메인(업로드) 화면

| 요소 | 설명 |
|------|------|
| 상단 | 서비스명 + 짧은 카피 |
| 손 선택 | [왼손] [오른손] 버튼 (필수) |
| 업로드 영역 | 파일 선택 + 드래그앤드롭 |
| 촬영 가이드 | 3줄 내외 안내 |
| 비저장 고지 | "업로드한 손바닥 사진은 결과 생성 후 즉시 폐기됩니다" |

### 6.2 대기(로딩) 화면

- 분석 중 로딩 표시
- 진행 상태 메시지

### 6.3 결과 화면

| 요소 | 설명 |
|------|------|
| 결과 카드 | 3D 오브젝트 중심 디자인 |
| 직업 타이틀 | 추천된 이색 직업명 |
| 해석 문장 | 2문단 (유머 + 근거) |
| 공유 버튼 | 링크 복사 |
| 다시하기 | 메인으로 이동 |

### 6.4 결과 랜딩 화면 (공유용)

| 요소 | 설명 |
|------|------|
| 결과 표시 | 결과 카드 + 해석 |
| CTA 버튼 | "나도 해보기" |
| 만료 안내 | 30일 경과 시 표시 |

---

## 7. 결과 카드 디자인 스펙

```
┌─────────────────────────────┐
│      [직업 타이틀]          │  ← 상단
│                             │
│    🎨 3D 오브젝트           │  ← 중앙 (핵심)
│       (직업 상징)           │
│                             │
│  ─────────────────────────  │  ← 손금 라인 패턴 배경
│                             │
│   "한 줄 코멘트" (선택)     │  ← 하단
└─────────────────────────────┘
```

- **스타일**: 3D 오브젝트 중심
- **배경**: 손금 라인 패턴 (은은하게)
- **목적**: 썸네일로 봐도 "직업 결과"가 즉시 이해됨

---

## 8. 문장 톤 가이드

- "단정"이 아니라 **"해석" 중심**
- **2문단 구성**:
  - 1문단: 가벼운 유머 (짧게)
  - 2문단: 그럴싸한 근거 (짧게)
- ❌ 불쾌감 유발 요소 배제 (혐오/성적/폭력/차별/불법)

---

## 9. 데이터 처리 정책

### 손바닥 사진 (⚠️ 중요)

| 항목 | 정책 |
|------|------|
| 저장 | ❌ 절대 저장 안 함 |
| 용도 | 결과 생성 목적 외 사용 금지 |
| 폐기 | 결과 생성 완료 후 즉시 |
| 장애 분석 | 저장 목적으로 사용 금지 |

### 결과물 보관

| 항목 | 내용 |
|------|------|
| 보관 대상 | 직업명, 결과 카드 이미지, 해석 문장 |
| 보관 기간 | 30일 |
| 만료 후 | "만료 안내 + 나도 해보기" 표시 |

## 11. 비용 최적화 전략

- ❌ 무제한 재생성/재시도 동선 미제공
- ✅ 실패 시 "촬영 가이드 기반 재업로드" 우선 유도
- ✅ 결과 카드 신규 생성 빈도 최소화
- ✅ 생성 실패 시 기본 카드/기본 결과로 대체

---

## 12. MVP 범위

### 포함 ✅

- 왼손/오른손 선택 + 업로드
- 결과 (직업/해석/결과 카드) 제공
- 저장/공유 (링크 중심)
- 결과 랜딩 페이지 + "나도 해보기" 전환
- 업로드 화면 간략 안내 (비저장/30일 유지/손 선택 유의)

### 제외 ❌

- 로그인/회원가입
- 개인 결과 보관함 (계정 기반)
- 랭킹/대전/커뮤니티 기능
- 유료 결제 기능
- 별도 FAQ 페이지

## 13. 성과 지표 (KPI)

| 지표 | 설명 |
|------|------|
| 완주율 | 업로드 대비 결과 도달률 |
| 공유율 | 결과 화면 저장/공유 클릭률 |
| 전환율 | 공유 랜딩 → "나도 해보기" 전환 |
| 실패율 | 손바닥 인식 실패/품질 문제 비율 |
| 생성 비율 | 결과 카드 신규 생성 비율 (비용) |

---

## 14. 광고 정책

- 업로드/결과 확인 핵심 행동을 방해하지 않는 위치
- 결과 카드보다 광고가 더 도드라지지 않도록 비중 통제
- 광고 지면 개수/위치는 MVP 화면 확정 후 결정
